Über das Programm

Dieses Programm liest eine .ESM - Datei, wie sie von Digitaltachographen oder den dazugehörigen Fahrerkarten verwendet wird, ein und wandelt sie in menschenlesbare Form um. Als Ausgabeformate stehen dabei HTML, XHTML oder einfacher Klartext zur Verfügung.

Digitaltachographen sind in der EU für Lastwagen mit einer zulässigen Gesamtmasse von mehr als 3.5 t vorgeschrieben. Sie speichern in digital signierter Form die Lenk- und Ruhezeiten, den Kilometerstand zu bestimmten Zeiten, Was bei Kontrollen geschieht und einiges mehr.

Um mit readESM die in Tacho oder Fahrerkarte gespeicherten informationen auszuwerten, müssen Sie diese zunächst herunterladen. Dies kann readESM nicht machen. Haben Sie dann die Datei, können Sie sie mit readESM in ein lesbares Format umwandeln. Die Bildschirmaufnahmen geben einen Überblick, was an Auswertung von readESM zu erwarten ist.

readESM ist freie Software - frei wie in freier Rede, und frei wie in Freibier. Sie dürfen readesm kostenlos herunterladen, und es nach den Bestimmungen der GPL verändern und weitergeben.

Installieren

Je nach verwendetem Betriebssystem bieten sich verschiedene Installationsmöglichkeiten. Leider habe ich bis jetzt noch keine Windows-Version erstellen können, doch sollte es prinzipiell möglich sein eine solche zu erstellen.

Ubuntu Linux

Veröffentlichte Versionen von readESM sind im meinem persönlichen Paketarchiv verfügbar. Dort finden sie auch detaillierte Informationen zum Einbinden des Archivs in ihr System. In Kurzform:

  • Fügen Sie mit 
    sudo nano /etc/apt/sources.list
    oder ihrem bevorzugten Paketmanager den Paketquellen das ppa hinzu: 
    deb http://ppa.launchpad.net/k-2005/ppa/ubuntu jaunty main
  • Installieren Sie readESM:
    sudo apt-get update
sudo apt-get install readesm
  • Wählen Sie in ihrem bevorzugten Dateimanager als Programm zum öffnen von ESM-Dateien readesm-wrap-kde oder readesm-wrap-firefox aus.
  • Neue Versionen werden automatisch durch update-manager oder sudo apt-get dist-upgrade installiert.

Kompilieren für andere Systeme

Sie können readESM-quelldateien von Sourceforge, meiner homepage, mittels Subversion oder als Quellpaket aus meinem PPA beziehen.

Um das Programm zu übersetzen, benötigen Sie einen C++ Kompiler, Teile der Boost-Bibliothek (nämlich program_options und shared_ptr), die Gnu MP-Bibliothek und libgrypt. Die letzteren beiden Abhängiglkeiten dienen dem prüfen der kryptographischen Signaturen in den Tachodateien.

Sobald alle Abhängigkeiten installiert sind, sollten Übersetzen und Installieren mit make und make install gelingen.

Um unter Ubuntu die aktuellste Version direkt aus dem Subversion-Baum zu übersetzen, gehen Sie wie folgt vor:

svn co https://readesm.svn.sourceforge.net/svnroot/readesm readesm
sudo apt-get install libboost-program-options-dev libgcrypt11-dev libgmp3-dev
cd readesm
make
sudo make install

Statt make install können Sie auch make packageinstall verwenden, es verwendet Checkinstall um ein Paket zu erstellen.

readESM benutzen

Unter den meisten Umständen werden Sie das Programm von der Kommandozeile wie folgt benutzen:

readesm --infile foo.esm --outfile bar.html --format=html

Sofern keine Ausgabedatei angegeben wird, wird stdout verwendet.

Die beiden Skriptdateien readesm-wrap-kde und readesm-wrap-firefox können verwendet werden, um ESM-Dateien umzuwandeln und die entstehenden Dateien in konqueror oder firefox anzusehen. Temporäre Dateien werden dabei in /tmp abgelegt. Die beiden Skripte sind insbesondere sinnvoll, um in einer grafischen Benutzeroberfläche wie KDE oder Gnome die Dateien anzusehen.

Veränderungen des Programms

Sie dürfen das Programm unter den Bedingungen der GPL verändern. Wenn sie dies tun, würde ich mich sehr freuen, wenn sie mir die veränderte Version zukommen lassen würden. Wenn Sie das Programm verändern wollen, beachten Sie:

  • Referenzen auf l207.pdf in den Kommentaren beziehen sich auf OJ:L:2002:207, p. 1-252. In dieser Verordnung steht vom Dateiformat der Tachographendaten bis zur elektrischen Verbindung beim Auslesen so ziemlich alles, was man an Informationen über Digitaltachographen benötigen könnte. Leider ist es nicht sehr einfach zu lesen. Die Verordnung ist in mehreren Sprachen verfügbar, Seitenangaben beziehen sich auf die englischsprachige Version. Die deutsche Version finden sie hier, für andere Sprachen das EN oder DE durch andere Ländercodes ersetzen.
  • Der Quellcode ist sehr unsauber, oft viele Klassen in der gleichen Datei, wilde Namenskonverntionen. Variablen sind oft nach den Vorgaben oben genannter Verordnung benannt, was zu langen Konstrukten wie controlDownloadPeriodBegin führt.
  • Mit Hilfe von make doc können sie eine Doxygen-Dokomentation erstellen. Eine solche ist auch online verfügbar.
